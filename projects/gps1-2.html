What is pair programming?

Pair programming is working with another developer to create code. Generally, only one person is typing at a time.

Why is pair programming important to DBC?

Pair programming is important to DBC for a couple of reasons. Pair programming is good practice for the real world where many developers work in teams. Pair programming is also important as a learning tool because working in groups enriches everyone's learning.

Comparing Driver and Navigator roles

The driver of the pair is the one who is actually typing the code into the computer. The navigator is the one who is helping to guide the process in the right direction.
