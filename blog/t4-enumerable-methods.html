Cycle

An enumerable method

2/3/15

The enumerable module contains many methods that can be applied to collections, i.e. arrays and hashes. One of these methods is called cycle. The syntax for cycle is:

collection.cycle(n){|loc_var| block}

When you call cycle on a collection, the collection is cycled through n number of times. Whatever is in the block is applied to each item of the list as its cycled through.

Examples

I'm pretty hungry, so let's create an array called "food" and fill it with different kinds of food:

food = ["pasta", "pizza", "cheese"]

If we wanted to print this list twice, we would write:

food.cycle(2){|bite| puts bite}

which would result in:

pasta
pizza
cheese
pasta
pizza
cheese

Say we wanted to print each of these words twice in each line. We would write:

food.cycle(2){|bite| puts bite*2}

which would result in:

pastapasta
pizzapizza
cheesecheese
pastapasta
pizzapizza
cheesecheese

Notice that the fourth line is "pizzapizza", not "pizzapizzapizzapizza". This method is non-destructive; the changes made to the items are not made to the original collection. The items from the original collection are cycled through each time. If this method were destructive, we would have in the fourth line "pizzapizza"*2 or "pizzapizzapizzapizza".

(Notes)

Here are some things to keep in mind about n:

  • If the integer entered is 0 or a negative number, nothing happens.
  • If a float is entered, the float will be rounded down to the nearest integer and the collection will be cycled through that amount of times.
  • If nothing is entered, the collection will be cycled through infinitely.
