Ruby Classes

2/13/15

Humans love to classify things. We like to be able to point to a whole bunch of different stuff that has some things in common and call it by one word. Take dogs for example. There are many breeds of dogs in the world. These breeds have many differences such as temperament, color, and size. However, because these breeds have certain attributes in common, such as four legs, wet noses and being adorable, there is a word in English to classify them all: “dog”.

Humans also love to classify things in computer programming. This is where “class” in Ruby comes in. Creating a class in Ruby is an easy solution for creating several objects with similar attributes.

For example, let’s create a Dog class in Ruby:

class Dog
    end

Now that we’ve created a Dog class, we can create a new Dog like so:

shih_tzu = Dog.new

To set inherent values to our Dog as soon as we’ve created it, we must enter values into the “initiate” method in our Dog class. These attributes will be assigned to all objects who are in class Dog. So let's make all of our dogs have 4 legs, wet noses and adorable like so:

class Dog
        def initiate
            legs = 4
            nose = “wet”
            adorable? = true
        end
    end

Now anytime we create a new dog, they will automatically have the attributes mentioned above.

But what about things that we want all dogs to have but need to be a little different? Like their names? One of the ways to solve this is to create a "name" variable and set it as an argument when we're defining initiate: 

class Dog
        def initiate(name)
            legs = 4
            nose = “wet”
            adorable? = true
        end
    end

We can now enter a name as an argument when we create a new dog object:

dalmatian = Dog.new(“Stephen”)

And we now have a dalmatian named Stephen. We’re living the dream.

On top of the attributes we assign to things in a class, we also can think of classes in terms of what we do to them. There are things that we can do to dogs like walk them, praise them, and dress them up like dragons. This is where methods come in. Methods inside of class definitions are the things we can do to our objects and/or dogs.

Let’s make a praise method for our class that says “Good boy” and then the name of our dog. In order to use the name we entered earlier, we’ll need to make the name available to our whole class by making it into an “instance variable”. Instance variables start with an @ symbol and are available for use within the whole class, not just within the current method.

class Dog
        def initiate(name)
            @name = name
            legs = 4
            nose = “wet”
            adorable? = true
        end
        def praise
            puts “Good boy, #{@name}!”
        end
    end

We can call our praise method on our dog Stephen by entering:

dalmatian.praise

which would result in:

Good boy, Stephen!

Thank you for reading this post! I hope it helped to clear up some things on classes.
