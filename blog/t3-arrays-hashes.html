Hashes and Arrays

1/29/15

In the programming language Ruby, Arrays and Hashes are used to store information. The main way that these two organizational tools differ is in how they let us retrieve information. When choosing whether or not to use an Array or a Hash in your code, it is important to consider how you want to retrieve your information.

Arrays

Arrays are organizational systems that are helpful for when you want your information to be retrievable based a numbered list. The syntax for writing an array is:

array = [“item0”, “item1”, “item2”, “item3”]

I am the youngest of four sisters. A standard thing that happens when people get to know me is they learn the names of my older sisters. Factoring in our bith order, the names of me and my sisters is information stored well in an array.

Here's an array with the names of me and my sisters stored in birth order:

milburn_sisters = [“Rebecca”, “Eleanor”, “Sarah”, “Debbie”]

In order to access information in an array, you type:

array_name[#]

with the corresponding number of the data you want where "#" is. An important thing to remember when accessing arrays is that the first piece of data stored is data number 0, NOT data number 1. So, if you wanted to find out the name of my oldest sister, you would type:

puts milburn_sisters[0]

which would result in:

Rebecca

Hashes

Hashes are organizational systems that are helpful for when you don't need your information stored in a specific order. The syntax for writing a hash is:

hash = {

“key1” => “value1”,

“key2” => “value2”,

“key3” => “value3”,

“key4” => “value4”

}

A dictionary is an example of an organizational system that does not require your information to be retrieved in a specific order. (I know dictionaries are usually in alphabetical order, but hear me out.) If you were to look up "Aardvark" in the dictionary, likely the only thing that would be important to you is the definition, NOT that Aardvark is the first word in the dictionary.

Hashes are usesful for this reason; they allow you to access information by their keywords, instead of by their place in a line.

Here's a hash dictionary involving the computer programming vocab in this blog post:

vocab = {

"hash" => "A data retrieval system, not ordered”,

"array" => "A data retrieval system, ordered",

"ruby" => "A programming language",

}

In order to access information in a hash, you type:

hash_name["key"]

with the word before the => where "key" is. So, if you wanted to find out what ruby is, you would type:

puts vocab["ruby"]

which would result in:

A programming language

I hope this was helpful in understanding the differences between Hashes and Arrays and when to use them.
