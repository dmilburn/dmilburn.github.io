Javascript Objects vs. Ruby Hashes

2/25/15

Creating Objects and Hashes

Objects in Javascript can either be defined with a single value or with several values. When Javascript Objects are defined with a single value, they resemble Objects in Ruby. This single value can be any data type.

In Ruby I would write:

full_name = "Harry Potter"

In Javascript I would write:

var fullName = "Harry Potter"

Objects in Javascript can also be defined with several values. When they are defined like this, they more resemble Ruby Hashes.

Here's a Javascript Object with several values:

var hp = {
 name: "Harry Potter",
 hairColor: "Black",
 eyeColor: "Green",
 house: "Gryffindor"}

There are two ways to write Ruby Hashes. In this blog, this way will be referred to as a colon hash:

hp = {
 name: "Harry Potter",
 hair_color: "Black",
 eye_color: "Green",
 house: "Gryffindor"}

In this blog, this way will be referred to as a rocket hash:

hp = {
 "name" => "Harry Potter",
 "hair_color" =>  "Black",
 "eye_color" =>  "Green",
 "house" => "Gryffindor"}

As you can see, the syntax for creating Javascript Objects is almost the same as a colon hash in Ruby. The only difference is the "var" at the beginning.

Accessing Values

There are two ways to access existing values in Javascript Objects. To have "Harry Potter" printed to the screen, I could either write:

console.log(hp.name)

or

console.log(hp["name"])

There are also two ways to access values in Ruby Hashes depending on which way you wrote your hash initially. If you use created a rocket hash, you write:

puts hp["name"]

If you used a colon hash, you write:

puts hp[:name]

Notice how the rocket hash syntax is more similar to Javascript object syntax in this case.

Creating New Values

To create a new value in Javascript Objects, you always write:

hp["scar"] = "Lightening Bolt"

There are two ways to create new values in Ruby Hashes depending on your initial hash type. For rocket hashes, you write:

hp["scar"] = "Lightening Bolt"

And for colon hashes, you write:

hp[:scar] = "Lightening Bolt"

Notice again how rocket hash syntax is identical to Javascript object syntax.

Summary

As you have seen, Javascript Objects and Ruby Hashes have some similarities syntactically. Coming from a Ruby perspective, it is important to remember how Javascript Objects contain syntax from both kinds of Ruby Hashes.
